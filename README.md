# 컴푸터싸인펜의 커스텀 블로그 테마

[여기를 눌러서 체험해 보세요](https://fr0mhellsmath.blogspot.com/)

## ✨ 핵심 기능 (Features)

### 🎨 모던 UI & 레이아웃 (Modern UI/Layout)
* **완벽한 다크/라이트 모드 지원:** 운영체제(OS)의 시스템 설정에 맞춰 자동으로 전환되며, 화면의 테마 토글 버튼을 통해 수동으로도 부드럽게 전환할 수 있습니다.
* **최적화된 3-Pane 구조:** 데스크탑 환경에서는 좌측 사이드바, 중앙 본문, 우측 플로팅 목차(ToC)가 조화롭게 배치되어 넓은 화면을 100% 활용합니다.
* **집중 모드 (Zen Mode):** 오직 글 읽기에만 몰입하고 싶을 때, 클릭 한 번으로 사이드바와 목차를 숨기고 본문 영역을 화면 중앙으로 넓게 확장합니다. (pc only)
* **스켈레톤 로딩 (Skeleton Shimmer):** 페이지가 로드될 때 덜컹거리는 현상(Layout Shift)을 방지하기 위해 뼈대 UI가 먼저 부드럽게 표시됩니다.
* **무한 캐싱 '더보기' 버튼:** 구식 페이징 방식을 버리고 '더보기' 버튼을 도입했습니다. 특히 브라우저 세션 캐싱을 활용하여, 뒤로 가기를 누르거나 페이지를 이동해도 기존에 불러온 글 목록과 스크롤 위치가 날아가지 않고 그대로 유지됩니다. (용량 초과 방지 안전망 내장)
* **스마트 헤더 & 진행률 바:** 글을 읽으며 아래로 스크롤하면 상단 헤더가 방해되지 않게 숨고, 위로 올리면 다시 나타납니다. 화면 최상단에는 현재 글을 얼마나 읽었는지 알려주는 프로그레스 바가 작동합니다.
* **이미지 줌 (Medium Zoom):** 본문에 삽입된 이미지를 클릭하면 화면 중앙으로 부드럽게 확대되어 디테일한 확인이 가능합니다.

### 🧮 수학 및 개발자 특화 (Math & Code)
* **MathJax v3 완벽 통합:** 빠르고 가벼운 최신 MathJax 엔진을 탑재했습니다. `$ ... $` (인라인 수식) 및 `$$...$$` (디스플레이 수식) 문법을 완벽하게 지원하며, 모바일 화면에서 긴 수식이 잘리지 않도록 수평 스크롤이 자동으로 적용됩니다.
* **1-Click Overleaf 연동:** LaTeX 코드 블록을 작성하면 우측 상단에 `↗ Overleaf` 버튼이 나타납니다. 코드를 클릭 한 번으로 컴파일 가능한 상태(논문 뼈대 자동 생성)로 묶어 Overleaf 새 창으로 즉시 내보냅니다.
* **고급 코드 하이라이팅:**
  * 개발자에게 친숙한 모노스페이스 폰트(JetBrains Mono 등)가 적용된 가독성 높은 코드 블록을 제공합니다.
  * 원클릭 복사 버튼을 지원하며, 복사 성공 시 직관적인 체크마크로 부드럽게 상태를 알려줍니다.
* **논문 폰트 모드:** 설정 패널에서 '논문(LaTeX)' 폰트를 선택하면, 본문 전체가 실제 영문/국문 논문을 읽는 듯한 클래식한 명조체 텍스트로 변환됩니다.

### 🪨 옵시디언 완벽 이식 (Obsidian Integration)
에디터 환경(Obsidian)과 퍼블리싱 환경(Blog)의 격차를 0에 가깝게 줄였습니다.
* **수학/증명 자동 넘버링:** `[!theorem]`, `[!definition]`, `[!proof]` 등의 수학적 콜아웃을 작성하면, 자바스크립트가 본문 구조를 분석하여 '정리 1.', '정의 2.' 형식으로 자동으로 번호를 매겨줍니다.
* **플로팅 팝업 각주:** 클릭하면 화면 맨 밑으로 튕겨 내려가는 답답한 각주가 아닙니다. 본문의 각주 번호를 클릭하면 그 자리에서 즉시 말풍선 팝업(Lightbox)이 떠올라 내용을 보여줍니다.
* **멀티컬러 형광펜:** 옵시디언의 기본 형광펜(`==텍스트==`) 문법을 지원하며, HTML에서 커스텀 색상을 입힌 형광펜도 원래 색상 그대로 화면에 유지됩니다.
* **플로팅 목차 (Floating ToC):** 본문에 작성된 H2~H4 헤딩을 자동으로 파싱하여, 스크롤을 따라다니는 우측 목차를 생성합니다. 드래그해서 위치를 옮기거나 클릭해서 접어둘 수 있습니다.

### 📚 시리즈 네비게이션 (Series Navigation)
* 연재물 작성을 위해 특정 태그(라벨)를 공유하는 글들을 하나의 시리즈로 묶어줍니다.
* 글 하단에 **'이 시리즈의 다른 글 모아보기' 드롭다운 메뉴**와 함께 **'이전 편 / 다음 편'으로 이동할 수 있는 네비게이션 버튼**이 자동으로 생성됩니다.




## 🚀 설치 및 초기 세팅 (Installation & Setup)

Blogger 플랫폼의 특성상 테마 코드를 덮어씌우는 것 외에도 모바일 환경을 위한 필수 설정이 필요합니다. 아래 순서를 정확히 따라주세요.

### 1. 테마 적용하기
1. 블로그 관리자 대시보드에 접속합니다.
2. 좌측 메뉴에서 **테마(Theme)**를 클릭합니다.
3. 현재 사용 중인 테마 우측의 🔽 화살표를 누르고 **백업(Backup)**을 눌러 만약의 사태를 대비해 기존 테마를 저장해 둡니다.
4. 다시 🔽 화살표를 누르고 **HTML 편집(Edit HTML)**을 클릭합니다.
5. 에디터 내부를 클릭한 뒤 `Ctrl + A` (Mac은 `Cmd + A`)를 눌러 전체 코드를 선택하고 `Delete` 키로 완전히 삭지합니다.
6. 다운로드한 본 테마의 `theme.xml` 코드를 전체 복사하여 빈 공간에 붙여넣습니다.
7. 우측 상단의 💾 **저장 아이콘**을 클릭합니다.

### 2. 모바일 테마 덮어쓰기 방지 (필수 🌟)
Blogger는 구형 모바일 기기를 위해 자체적인 모바일 테마를 강제로 덧씌우는 기능이 있습니다. 본 테마는 이미 완벽한 반응형(Responsive)으로 제작되었으므로, 이 구형 덮어쓰기 기능을 반드시 꺼주어야 모바일에서 UI가 깨지지 않습니다.
1. 관리자 메뉴의 **테마(Theme)** 페이지로 돌아옵니다.
2. '맞춤설정' 옆 🔽 화살표를 누르고 **모바일 설정(Mobile Settings)**을 클릭합니다.
3. "모바일 기기에서 모바일 테마를 표시하시겠습니까?" 라는 질문에 **데스크톱(Desktop)**을 선택합니다.
4. **저장**을 누릅니다. 이제 데스크탑과 모바일 모두에서 모던한 UI가 동일하게 출력됩니다.

### 3. 화면 설정 (Settings Panel)
블로그 메인 화면에 접속한 뒤 우측(또는 우측 하단)의 `⚙ 톱니바퀴` 아이콘을 눌러 초기 환경을 세팅합니다. 이 설정값은 방문자의 브라우저(로컬 스토리지)에 안전하게 저장됩니다.
* **테마 모드 (Theme):** `Auto(시스템 연동)`, `Light`, `Dark` 중 선택.
* **글꼴 설정 (Font):** 기본 고딕체(Pretendard)와 수학/논문용 명조체(LaTeX Style) 중 블로그 성격에 맞는 폰트를 고정할 수 있습니다.

## 📝 글쓰기 워크플로우 1: 블로그 자체 에디터 (Native Blogger Editor)

옵시디언 같은 외부 에디터를 거치지 않고, Blogger 자체 에디터에서 직접 글을 작성할 때의 워크플로우입니다. 복잡한 마크다운 없이도 테마의 모든 강력한 기능을 사용할 수 있습니다.

### 1. 컴포즈 모드와 HTML 모드 활용
Blogger 에디터 좌측 상단의 연필 아이콘(`✏️` / `<>`)을 눌러 두 가지 모드를 오갈 수 있습니다.
* **작성(Compose) 모드:** 일반적인 글, 이미지 첨부, 굵기/색상 조절 등 뼈대를 잡을 때 사용합니다.
* **HTML 모드:** 코드 블록에 클래스를 입히거나, 정교한 HTML 태그를 넣을 때 사용합니다.

### 2. 수식 (MathJax) 작성하기
수식은 작성(Compose) 모드나 HTML 모드 어디서든 그냥 텍스트 치듯이 자연스럽게 입력하면 됩니다. 테마가 알아서 렌더링합니다.
* **인라인 수식:** 문장 중간에 수식을 넣을 때는 `$ ... $` 기호로 감쌉니다.
  * *입력:* `질량-에너지 등가 원리는 $E=mc^2$ 입니다.`
* **디스플레이 (블록) 수식:** 중앙에 정렬되는 큰 수식을 넣을 때는 `$$...$$` 로 감쌉니다.
  * *입력:* `$$\int_{a}^{b} f(x) dx = F(b) - F(a)$$`

### 3. 옵시디언 콜아웃 (Callout) 사용하기
Blogger 에디터의 **작성(Compose) 모드**에서 상단 툴바의 **'인용(Quote)' 버튼**을 활용합니다.
1. 에디터에서 인용(Quote) 버튼을 눌러 인용구 블록을 생성합니다.
2. 첫 줄에 반드시 `[!info]`, `[!theorem]`, `[!warning]` 등 원하는 콜아웃 문법을 적습니다.
3. 접기/펴기 기능을 쓰고 싶다면 `[!info]-` 처럼 뒤에 `-` 기호를 붙입니다.
4. 그 아랫줄부터 내용을 작성합니다. (발행하면 예쁜 박스로 자동 변환됩니다.)

### 4. 코드 블록과 Overleaf 연동
코드 블록은 **HTML 모드**에서 작성하는 것이 가장 안전하고 깔끔합니다.
```html
<pre><code class="language-python">
def hello_world():
    print("Hello, computerpen!")
</code></pre>
````

- **Overleaf 연동:** 언어를 `language-latex` 또는 `language-tex`로 지정하기만 하면, 발행 시 해당 코드 블록 우측 상단에 자동으로 **[↗ Overleaf]** 버튼이 생성됩니다.
    

### 5. 연재물 (시리즈) 설정하기

특정 글들을 묶어서 글 하단에 '시리즈 모아보기' 네비게이션을 띄우려면 **라벨(Tags)** 기능을 사용합니다.

1
1. 글 제목이나 글 내부의 h1 해딩을 숫자-숮자-제목 형식으로 작성합니다 (ex: 01-02-제목)
2. 글 작성 화면 우측 사이드바의 **라벨(Labels)** 입력칸을 클릭합니다.
3. 시리즈 이름으로 쓸 단어(예: `[선형대수학 기초]`, `딥러닝_튜토리얼`)를 입력합니다.
4. 연재할 다음 글들에도 **정확히 똑같은 라벨+정확한 제목**을 달아주면, 발행 순서에 따라 자동으로 1편, 2편, 3편... 으로 묶여 인덱싱됩니다.



## 📝 글쓰기 워크플로우 2: Obsidian ➔ Blogger 완벽 연동

이 테마는 옵시디언에서 작성한 마크다운 문서가 HTML로 변환될 때 생기는 구조(특히 Pandoc 변환 구조)를 자바스크립트가 자동으로 인식하고 보정하도록 설계되었습니다. 따라서 일일이 코드를 수정할 필요 없이 아주 매끄러운 퍼블리싱이 가능합니다.

### 1단계: 옵시디언에서 자유롭게 글쓰기
평소 옵시디언을 사용하시던 방식 그대로 문서를 작성하세요. 테마가 알아서 처리해 주므로 문법을 타협할 필요가 없습니다.
* **수식:** `$E=mc^2$` 처럼 평소대로 입력합니다.
* **콜아웃:** `> [!theorem]` 처럼 꺾쇠(`>`)와 대괄호를 사용해 입력합니다.
* **형광펜:** `==중요한 내용==` 처럼 등호 2개를 사용합니다.
* **각주:** `문장 끝에 각주를 답니다.[^1]` 처럼 옵시디언 기본 각주 문법을 사용합니다.
* **코드 블록:** 백틱 3개(```)와 함께 언어(python, cpp, latex 등)를 명시해 줍니다.

### 2단계: HTML로 변환하기 (Pandoc 등 활용)

블로거는 마크다운을 직접 지원하지 않으므로, 옵시디언의 마크다운을 HTML로 변환하는 과정이 필요합니다. 가장 추천하는 방식은 **Pandoc 기반의 플러그인**을 사용하는 것입니다. (예: `Obsidian Pandoc Plugin` 또는 `Copy as HTML` 커뮤니티 플러그인)

* **Pandoc 변환 시 장점:** 각주가 블로그와 호환되는 완벽한 앵커 태그(`<a href="#fn1">`)로 변환되며, 코드 블록이 `<pre class="sourceCode python">` 형태로 깔끔하게 묶입니다.

### 3단계: Blogger에 붙여넣기 및 자동 변환 (Magic!)
1. 변환된 HTML 코드를 전체 복사합니다. (만약 `<body>` 태그가 포함된 전체 HTML 파일로 출력되었다면, `<body>` 안쪽 내용만 복사하세요.)
2. Blogger 글쓰기 화면으로 들어가 **[HTML 모드]**로 전환합니다.
3. 복사한 HTML을 그대로 붙여넣기 하고 **발행(또는 미리보기)**을 누릅니다.

**💡 붙여넣는 순간, 테마 내부에서 일어나는 자동 최적화:**
* **Pandoc 코드 블록 보정:** Pandoc이 만들어낸 투박한 `sourceCode` 태그를 테마가 자동으로 감지하여, Prism.js가 인식할 수 있는 `language-python` 등의 깔끔한 툴바 박스로 실시간 변환해 줍니다.
* **옵시디언 각주 팝업화:** 문서 맨 밑에 생성된 각주 원본 리스트를 시각적으로 깔끔하게 숨기고, 본문 속 각주 번호를 누를 때마다 예쁜 팝업(Tooltip)으로 띄워줍니다.
* **빈 코드블록 에러 방지:** 언어가 지정되지 않은 코드 블록이 깨지는 현상을 막기 위해, 테마가 알아서 `language-none` 꼬리표를 달아 완벽한 까만 박스로 보호해 줍니다.
